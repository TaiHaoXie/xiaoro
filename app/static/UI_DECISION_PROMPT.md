# 决策过程可视化UI优化Prompt

## 给Gemini的完整Prompt（直接复制）

---

```
你是一位专注于B2B数据分析可视化的高级UI设计师。请优化以下"决策过程可视化"组件的设计。

## 当前问题
- Emoji图标显得不够专业
- 整体设计偏消费级，缺乏数据产品的严谨感
- 需要体现"技术透明"和"可复现"的工程价值

## 设计目标
打造类似 Linear/Notion/Stripe 的专业级数据可视化组件风格：
- 简洁、克制、信息密度高
- 使用几何图形替代Emoji
- 强调数据逻辑和决策链条
- 莫兰迪色系保持一致

## 组件结构
决策过程包含5个步骤，每个步骤需要展示：
1. 步骤编号（01, 02, 03...）
2. 步骤标题（如"需求分析"）
3. 步骤描述（简短说明）
4. 关键数据（可折叠）
5. 置信度/匹配度（进度条或百分比）

## 设计要求

### 图标系统
- 用几何图形替代Emoji：
  - 圆形 = 分析/理解类
  - 菱形 = 决策/判断类
  - 正方形 = 数据/事实类
  - 三角形 = 推荐/输出类
- 图标大小：16x16px
- 颜色：莫兰迪蓝灰 (#8B9DC3)

### 步骤卡片
```
┌─────────────────────────────────────────────┐
│  ○ 需求分析                    85% ████████░░ │
│  识别购物意图、预算范围、品牌偏好           │
│  ▼ 展开数据                                  │
└─────────────────────────────────────────────┘
```

- 背景：白色
- 边框：1px solid #E8E6E3
- 圆角：8px
- 内边距：12px
- 卡片间距：8px
- 左侧连接线：2px dashed #D8CFC4

### 连接线
- 步骤之间用虚线连接
- 颜色：#D8CFC4
- 样式：dashed 2px
- 方向：从上到下

### 数据展开区域
- 背景：#F7F7F5
- 圆角：6px
- 字体：11px monospace（等宽字体）
- 数据项用 `|` 分隔

### 置信度展示
- 使用细进度条
- 高匹配(>80%): #9CAF88 (鼠尾草绿)
- 中匹配(60-80%): #8B9DC3 (灰蓝)
- 低匹配(<60%): #D8CFC4 (燕麦)

### 折叠/展开动画
- 高度过渡：0.3s cubic-bezier(0.4, 0, 0.2, 1)
- 箭头旋转：180deg
- 数据区域淡入：opacity 0→1

## 输出要求

请生成完整的HTML/CSS/JS代码，包括：
1. 5个步骤的完整示例数据
2. 折叠/展开交互
3. 响应式设计（移动端适配）
4. 符合现有莫兰迪色系变量

参考设计：
- Linear Pipeline View
- Notion Database
- Stripe Dashboard
```
